# 15: データフロー図 🔄

## プロジェクト作成のデータフロー

```
┌──────────────────────────────────────────────────────┐
│              プロジェクト作成のデータフロー           │
└──────────────────────────────────────────────────────┘

[Client]
  │ POST /api/projects { name: "プロジェクト名" }
  ↓
[API Route Handler]
  │ リクエストバリデーション
  ↓
[ProjectsService.createProject()]
  │ 1. バリデーション（Service層）
  │ 2. ProjectsRepository.create() 呼び出し
  ↓
[ProjectsRepository.create()]
  │ 1. バリデーション（Repository層）
  │ 2. PME番号自動生成
  │ 3. DB INSERT実行
  │ 4. 結果をProjectTable型にマッピング
  ↓
[Database: projects]
  │ INSERT実行
  ↓
[ProjectsRepository]
  │ ProjectTable型を返却
  ↓
[ProjectsService]
  │ ApiResponse型にラップして返却
  ↓
[API Route Handler]
  │ JSONレスポンス返却
  ↓
[Client]
  │ { success: true, data: {...}, message: "..." }
```
