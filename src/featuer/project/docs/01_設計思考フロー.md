# 01: 設計思考フロー 📋

## 設計思考フローに基づく分析

```
既存データベース
      ↓
【STEP 1】テーブル群を機能ドメインにグルーピング
      ↓
【STEP 2】各ドメインのユースケースを抽出
      ↓
【STEP 3】ユースケースからService境界を決定
      ↓
【STEP 4】Serviceが必要とするRepositoryを特定
      ↓
【STEP 5】Repository間の依存関係を整理
```
